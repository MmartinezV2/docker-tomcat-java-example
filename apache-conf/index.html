Apache & Tomcat Reverse Proxy Example

This is a demonstration page for educational use on Web Application Deployment at EsLiceu.com

By using the following links you will see the difference between a http reverse proxy balancing to a tomcat pool and to a tomcat routed pool using jvmRoute and Sticky

Examples based on the work of:

  • Maik Hilter - https://github.com/mhilker/docker-tomcat-java-example
  • Darren Danvers - https://codestijl.dev/2020/09/14/a-gradle-build-in-docker/
  • Chandan Kumar - https://geekflare.com/tomcat-load-balancer-using-mod-proxy-and-session-sticky/
  • Sample code from Tomcat9-Examples

Local links (supposing you are accessing them on localhost):

  • 1st Tomcat container (direct/no proxied): http://localhost:8091/
  • 2nd Tomcat container (direct/no proxied): http://localhost:8092/
  • 1st Tomcat container (proxied): http://localhost:8080/web1/
  • 2nd Tomcat container (proxied): http://localhost:8080/web2/
  • Tomcat Load Balancer Pool (without sticky sessions): http://localhost:8080/loadbalancer/
  • Tomcat Load Balancer Pool (using sticky sessions): http://localhost:8080/stickyloadbalancer/

Note that on the Routed Load Balancer Pool, once you (a client) stablish a session you will always be routed to the same Tomcat instance. This way, there's no need to use clustering features to share sessions between tomcat instances. You can use standalone tomcat instances.

Reload the web page on the loadbalancer links to see the efect of the use of sticky sessions
